# Help Desk Ticket Template

## Overview

This template can be used to document common help desk, service desk, desktop support, and technical support issues.

A strong ticket should clearly explain the user issue, what troubleshooting was performed, what was found, what action was taken, and how the issue was resolved or escalated.

## Ticket Information

| Field | Details |
|---|---|
| Ticket Number | HD-000000 |
| Date Opened | YYYY-MM-DD |
| Date Closed | YYYY-MM-DD |
| Request Type | Incident / Service Request |
| Priority | Low / Medium / High / Critical |
| Status | Open / In Progress / Resolved / Escalated / Closed |
| Category | Hardware / Software / Network / Email / Access / Printer / VPN |
| User | User Name |
| Department | Department Name |
| Device Name | Computer or Asset Name |
| Location | Office / Remote / Hybrid |

## Issue Summary

Short description of the issue.

Example:

```text
User is unable to access email in Outlook desktop application.
```

## User Impact

Describe how the issue affects the user or business process.

Example:

```text
User cannot send or receive email from Outlook desktop client, impacting customer communication.
```

## Initial User Report

Document what the user reported in their own words when possible.

Example:

```text
User stated that Outlook keeps showing disconnected and new emails are not appearing.
```

## Troubleshooting Questions

Ask relevant questions such as:

- When did the issue start?
- Is the issue affecting one user or multiple users?
- Is there an error message?
- Is the user working remotely or onsite?
- Has anything changed recently?
- Does the issue happen on another device?
- Does the issue happen in webmail or only in the desktop application?

## Troubleshooting Steps Performed

Document each step clearly.

Example:

```text
1. Confirmed user identity.
2. Verified device name and operating system.
3. Tested internet connectivity.
4. Tested application in browser/web version.
5. Restarted application.
6. Checked account status.
7. Reproduced issue with user.
8. Applied fix and retested.
```

## Findings

Summarize what was discovered.

Example:

```text
Outlook desktop was disconnected, but Outlook on the web worked normally. This confirmed the mailbox was active and the issue was local to the Outlook desktop client.
```

## Resolution

Document what fixed the issue.

Example:

```text
Created a new Outlook profile, re-added the user's Microsoft 365 account, and confirmed email sync was restored.
```

## Verification

Confirm the issue was resolved.

Example:

```text
User confirmed Outlook opened successfully, new emails appeared, and test email was sent and received.
```

## Escalation Notes

Use this section if the issue requires another team.

Example:

```text
Escalated to Microsoft 365 administrator for message trace review because the issue may involve mail flow rules or quarantine.
```

## Final Ticket Note

Use a clear closing note.

Example:

```text
User reported Outlook disconnected and not syncing email.
Confirmed Outlook on the web worked successfully.
Restarted Outlook and checked network connectivity.
Created a new Outlook profile and reconnected the Microsoft 365 mailbox.
Confirmed email sync, send, and receive functionality.
User verified issue was resolved.
Ticket closed.
```

## Best Practices

- Keep notes professional and factual.
- Avoid blaming the user.
- Record important error messages.
- Include device name and affected application.
- Document troubleshooting steps in order.
- Include the final resolution.
- Escalate clearly when needed.
- Do not include passwords in ticket notes.
- Protect sensitive user information.
- Use clear language that another technician can understand.
